Defining and running functions

Reusable procedures that allow us to write reusable, modular code

Defining a function:

function funcName() {
    // do something
}

Running/executing a function:

funcName()

All methods are functions, more on that later.

Arguments

We can also write functions that accept inputs, called arguments

Adding inputs to a function:

function funcName(input, input2, etc) {
    // do something
}

When we create the function, the inputs we require are called parameters, but when we execute/run the function are then called arguments.

Example

// Defining the function
function greet(firstName) {
    console.log(`Hey there, ${firstName}`);
}

// Running the function
greet("Carlos"); // Hey there, Carlos

We can input many arguments, but we must pay attention to the order of the parameters when we define the fuction.

Multiple arguments

function repeatThing(thing, numTimes) {
    let resultStr = "";
    for (let i = 0; i < numTimes; i++){
      resultStr += thing;
    }
    console.log(resultStr);
  }
  
  repeatThing("lol", 5); //lollollollollol

The return keyword

Built-in methods return storable values when we call them.

Method returning a value

const yell = "Die bart, die!".toUpperCase();

console.log(yell); // DIE BART, DIE!

So far, our functions don't return a value (or return undefined) when called, they're just printing something to the console.

Return in action

function sum(x, y) {
    let total = x + y;
    return total;
    console.log("Honk"); // This line of code won't run dfgdagdsgsdgfsdfgdsfgdfgsdgdfg
}

const mathProdigy = sum(2, 2);
console.log(mathProdigy); // 4

On top of that, return, when encountered in a function will prevent any further code from executing (in the function).
